搭建博客

发表于 2017-01-03

借用hexo在git上搭建自己的博客

注意:以下凡是涉及到tu123dou都要改为你自己github上的名字

start

  • 新建仓库
  • 仓库名字为tu123dou.github.io
  • 打开cmd命令行->进入某个盘(随意选择位置)
  • 生成项目

需要进入您的博客文件夹里进行以下操作

  • 启动服务

  • 安装发布工具

  • 在 _config.yml 中写入下面:

  • hexo deploy

  • hexo g (再次发布时先执行 hexo g 在执行 hexo deploy)
  • 成功后 在网址输入tu123dou.github.io

    end

    博客主题
  • 选择一个自己喜欢的主题

  • 点Clone or download 进入您的博客文件夹->themes文件夹里->在这里克隆
  • 修改_config.yml文件里的 theme: [刚刚下载好文件夹的名字]
  • 为了方便修改一般都是先运行hexo s --debug或者hexo server可以在本地查看,满意后在hexo g+hexo deploy或者是hexo d -g

发布静态网页

发表于 2017-01-03

第一种方法->借用git发布网页

start

  1. 在github上创建一个仓库
    • 注意要勾选 Initialize this repository with a README
    • Settings -> GitHub Pages-> Choose a theme(随意选择一个模板或者专题)
  2. 把仓库地址克隆到本地
    • git init
    • git clone [仓库地址]
  3. 把要发布的网页文件放到上步的文件夹里

    注意: 如果访问页面是项目根目录下的index.html的话,我们访问的时候可以省略文件名不写,这样默认找的也是index.html,如果不是index则必须要写上才可以

  4. 把这个文件夹提交到github上
    • git add .
    • git commit -m”WebPage”
    • git push origin master
  5. 回到github的仓库,在setting里面会有网址
    • 一般网址格式:https://用户名.github.io/仓库名/index.html

      注意:github只能上传和发布前端静态页面(不能有后台程序,但是页面中的数据可以跨域访问实现动态绑定

      end

第二种方法->在gitHub上挂载静态页

  • 需要创建一个仓库
  • 挂载静态页必须将代码提交到名为gh-pages分支上
  • 将这个分支推送到github上
  • 在seeings中可以看网址
